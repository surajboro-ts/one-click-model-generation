# Color System

## **The color palette is used to create a unified and recognizable consistency in ThoughtSpot’s product and its array of features and interfaces.**

---

This consistency is grounded in a set of well-defined rules on how to work with color when organizing content, assigning semantic meaning to alerts or creating data visualizations, illustrations and interaction states.

## **Color tokens**

Radiant colors are managed using color tokens that abstract color by role or usage, independent of the actual color hex values. Design token components include built-in color calls. These components are architected with layers of abstraction so that, if global color changes become necessary, they’re easy to make.

| **Value** | The actual style (hex code) assigned to a token. | **#1D232F** |
| --- | --- | --- |
| **Generic token** | Color group of a token. | **Gray 90** |
| **Role token** | The systematic usage(s) of a token. | **Text & Icon / Default** |

## **Interaction states**

### **Component states**

Buttons are clickable elements that are used to trigger actions. They communicate calls to action to the user and allow users to interact with pages in a variety of ways.

Primary buttons have a default state and color. Other states such as hover, focus (for accessibility), active, or disabled can be differentiated by increasing or decreasing the darkness value by increments of 10.

e.g primary button state is blue 60, hover state is blue 70, Pressed or Active state is blue 80 and so on.

Focus states always have a 2px wide primary blue outline with a darkness value of 50. Note that all focus states have to be the same including forms or any other focused state items. For more detail on focus states, see Radiant’s accessibility guidelines.

## **Accessible colors**

Radiant adheres to WCAG AA web accessibility standards including a 4.5:1 color contrast between text and background for normal text, and 3:1 to large text.

WebAIM Web Accessibility in Mind provides more guidance for how to meet accessibility requirements with text on dark and light backgrounds, colored text,  helper text and selected text and more.

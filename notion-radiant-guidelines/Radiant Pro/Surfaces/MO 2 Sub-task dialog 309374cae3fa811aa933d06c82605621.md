### Examples

Ex.1 Filter modal

**User:** As a user I want to add a filter in my answer, so that I can get filtered chart. Therefore I go to the left rail and click on a column to edit it.

**System Response:** Click the checkbox to select the column.

Show the total selected number on the top. Clicking the summary checkbox here can select/deselect all columns.

Click “Add bulk values” button opens a subtask modal to add bulk values in the table.

Toggle ‘show selected’ to show selected values.

Click “Add” to close the dialog box and add value.

Click “Cancel” to close the dialog box and won’t add value.

**System Response:** opens a subtask modal to add bulk values in the table with text input.

helper text suggesting user to enter values with comma

Click ‘back’ to go back to the add filter modal.

Ex.2 SpotIQ wizard flow

Ex.2.1 SpotIQ wizard flow

**User:** As a user I want to use AI to analyse some columns of my answer therefore I go to the SpotIQ option and launch it. The modal opens asking me to select few desired columns for spotIQ to run the analysis.
**System Response:**
Click the checkbox to select an object.
Show the total selected number on the top. 
Clicking “select all” will select all columns.
Clicking “clear” will unselect all columns.

Toggle ‘show selected’ to show selected values.
Click “Cancel” to close the dialog box and won’t add columns.
Click “Previous” to go back to SpotIQ wizard.
Click “Analyse” to start the SpotIQ analysis of the columns.

Ex.3 Share flow

Ex.3.1 Share flow

**User:** As a user I want to share my liveboard with people inside my organisation using TS. Therefore I go to the share button in my answer and click on it.
**System response:** A modal with a add user and group search bar opens, along with an option to include a message with with the answer. There is option to share liveboard editor access and access to viewing data source. 

Manage lets us manage the users that have already access to this liveboard.
Cancel does not share and closes the modal.
Share, shares the liveboard and closes the modal.

**User:** As a user I want to share my liveboard with people inside my organisation using TS. Therefore I go to the share button in my answer and click on it. I then go to manage option to check how all have access to the liveboard.

**System response:** A modal with a  user and group search bar opens, below that we have list of users and groups who have access to the liveboard or if their request is pending.

Resend link, send invite link again to the user who didn’t accept the invitation yet.
Remove link, removes the user/group’s access to the liveboard.
Done, shares the liveboard and closes the modal.

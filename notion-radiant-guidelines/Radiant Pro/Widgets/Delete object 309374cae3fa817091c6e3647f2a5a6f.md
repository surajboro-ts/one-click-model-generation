# Delete object

**DL.0 Delete core pattern** 
The delete pattern allows users to remove an object or a configuration from the system. 
In some cases deleting an object can have additional effects on the system, so an extended warming message and reconfirming password fields are required. 

**User action:** The user needs to remove an object or configuration from the system 
**System response:** The system will display either a message confirming the object or configuration they are about to delete, and in some cases a reconfirmation field as a secondary precaution. The modal header will contain the word delete + “object or configuration name”

# Object Table

## **OL.0 Object List**

**Overview** 
The object list is designed to show tabular data. It is designed in two modes, read-only and editable. 

The object list allows the user to search, filter and sort the information in the tabular widget. 

**User Action**: Users want to search and filter object lists; they also may want to take specific actions on single or multiple objects. They can also select a single object to see the full object view page.

**System Response**: The system starts in a default state of all objects, no filters selected and one or two global actions that will affect the object list.

## **OL.4 Editable Object table**

**Overview**

Object Table allows users to enter information into it so they can update the values in the table
Object Table has a spreadsheet-like interface to allows a user to move between and select cells
The object table will have a different visual appearance. 

**User Action:** Object Row allows users to update and edit content inline

**System Response:** The system will let users either to enter in content directly or they can select from a predefined selection pulldown list.

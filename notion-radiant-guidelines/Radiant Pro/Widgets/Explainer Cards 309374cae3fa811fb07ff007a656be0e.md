# Explainer Cards

### **Definition**

Explainer cards provide a flexible format in which to answer the complete range of user questions. They can include titles, links, images, and buttons and multiple lines of text.

### **Explain card triggers**

With rare exceptions, explain cards are activated by a mouse click rather than a hover-over. Users can dismiss a explain card either by clicking the “x” button or clicking outside of the card.

Explainer icon at 14px x 14px [Click]

Text link [Click to trigger]

Icon btn at 32px / 24px / 20px [Click]

Text string [Hover w/ 500ms delay]

Disabled UI element [Hover w/ 500ms delay]

### **Content guidelines**

When a screen component, text element or content type requires more lengthy clarification or context, the user can activate an explain card AKA an **i** button.

Explain card should answer questions like “What is this?” or “What does this term mean?” or “How do I use this?”

Succinctly provide information for the specific term or feature.

**Writing explain cards**

- Include a 2 – 4 word title
- Use the title to add context “Types of joins” instead of “Joins”
- Use the UX writing pattern for long body text
- 40 characters per line for a maximum of 4 lines
- For large amounts of text, separate into 4 line blocks
- Explain cards can contain images, illustrations, or videos
- A title should be the first messaging on the card

Reference system

Priority reference files

The AI consults these 10 files in priority order. All live in .cursor/rules/ and are loaded automatically.

1
_orchestration.md
Router
Determines which guidelines to consult based on the task type
2
prototype-generation.md
Core hub
Code structure, import patterns, and generation workflow steps
3
component-inventory.md
Component picker
Decision tree to find the right component for each UI element
4
widget-patterns.md
Interaction rules
Alert types, menu ordering, delete confirmation, tooltips, tables
5
layout-patterns.md
Page templates
Dashboard, admin, form, table page, and wizard layouts
6
figma-component-mapping.md
Figma translator
Maps Figma layer names and screenshots to Radiant components
7
modal-patterns.md
Modal guide
Size selection (M1-M4), types, footer placement, wizard progress
8
token-usage.md
Styling ref
Color scales, spacing values, typography — never hard-code
9
content-guidelines.md
UI text rules
Button labels, titles, errors — ThoughtSpot content patterns
10
product-knowledge.md
Domain context
ThoughtSpot features: Answers, Liveboards, Spotter, SpotIQ
Reuse strategy

Component reuse

The AI always prefers existing Radiant components. New ones are only created when nothing suitable exists.

✅

Exact match

Imports directly from the shared component library

Preferred
🔧

Close match

Uses existing component with custom props or styling overrides

Common
🆕

No match

Creates a local component in your prototype folder

Last resort
💡 Local components follow the same design-system rules (tokens, forwardRef, TypeScript) but live inside your prototype folder, keeping the shared library clean.

Now you know the system

How Git works

Your work lives on its own branch

The main Radiant Play repo is always safe. You branch off, build your prototype, then merge back when it's ready — or just keep it on your branch forever.

main branch
The live Radiant Play repo on Galaxy. Always stable. Only changes after a reviewed Pull Request is merged in.
your branch
Your personal workspace. Invisible to everyone until you open a PR. Build, break, experiment — main is never affected.

Before you start

Two things to set up

One-time setup. Takes about 5 minutes total.

1

Galaxy account

Your ThoughtSpot login works here. Sign in and confirm you can see the Radiant Play repo. Can't access it? Message @faris on Slack.

galaxy.corp.thoughtspot.com
2

Cursor IDE

The AI editor where you build prototypes. Download it, then create a free account at cursor.so to activate it. Handles git and npm automatically. Mac and Windows.

cursor.so
Step 1

Fork the repo

A fork is your personal copy. Changes you make only affect your version — never the original.

1

Go to Radiant Play on Galaxy

Find the Radiant Play repository on Galaxy

2

Click "Fork" in the top right

Galaxy creates your personal copy under your account — takes ~10 seconds

3

Copy your fork's clone URL

On your fork page, click the green Code button → copy the HTTPS URL. It looks like:
https://galaxy…/your-name/radiantplay.git

Step 2

Get it running on your machine

Steps 1–3 are one-time only. After that, only "Start the server" each time you come back.

Clone in Cursor
Two ways to do this in Cursor:
Option A: File → Clone Git Repository
Option B: Click the branch icon in the left sidebar → "Clone Repository"
Paste your fork URL, choose a folder (Desktop is fine). Cursor downloads the project.
# Paste your fork URL — looks like this: https://galaxy.corp.thoughtspot.com/your-name/radiantplay.git # Replace "your-name" with your Galaxy username
Open the terminal inside Cursor
Go to Terminal → New Terminal in Cursor's menu bar. A panel opens at the bottom — this is where you type commands.
# Keyboard shortcut (faster): Ctrl + ` # that's the backtick key, top-left of keyboard
Install dependencies one-time only
Downloads all the packages the project needs. Takes ~1 minute. You'll never run this again.
⚠ If this fails: Node.js may not be installed. Run node --version to check — if nothing shows, install it from nodejs.org then retry.
npm install added 76 packages in 54s ✓
Start the dev server every session
Run this each time you open the project. Leave the terminal running — it auto-updates the browser when you make changes.
npm run dev VITE v7 ready in 312ms ➜ Local: http://localhost:5173/ # Two URLs to know: localhost:5173 ← Radiant DS browser (components, tokens) localhost:5173/playground← YOUR prototypes live here ✓
Step 3

Build your first prototype

Describe what you want. Cursor knows all 73+ components and ThoughtSpot patterns.

⌨️

Create a new prototype

In terminal: npm run new-prototype SettingsPage
Must start with a capital, no spaces. ✓ SettingsPage   ✗ settings-page

📂

Open the generated file first

In Cursor's file tree: src/prototypes/SettingsPage/index.tsx
Cursor AI needs a file open to know where to write the code

💬

Open AI chat and describe your UI

Cmd+L (Mac) / Ctrl+L (Windows) — type a description, or drag a screenshot into the chat and say "Recreate this using Radiant components"

🔗

Share your prototype

Anyone on the same WiFi can open your URL locally at localhost:5173/playground. To share remotely, push to your Galaxy fork.

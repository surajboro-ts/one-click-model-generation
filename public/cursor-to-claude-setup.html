Claude setup and skills

A one-time setup to get Claude Code running, plus the skills available to help you build prototypes.
Time ~5 minutes
Steps 6 steps
Requires Terminal + internet
↻ Keep your Radiant Play library up to date. New components and skills may be available. Update library →
1

Install Claude Code

Official one-line installer from Anthropic — handles everything automatically

Open your terminal and run:

Terminal
$ curl -fsSL https://claude.ai/install.sh | bash

Once the installer finishes, reload your shell so the claude command is recognised:

Terminal
$ source ~/.zshrc

Verify the installation worked:

Terminal
$ claude --version
# Should print a version number, e.g. 1.x.x
✅ If a version number prints, Claude Code is installed. Move on to Step 2.
⚠️ command not found: claude? The installer may not have updated your PATH. Run source ~/.zshrc again, or open a new terminal window and retry.
2

Create your Claude account and API key

You'll receive an email invite — this takes about 2 minutes
  • 1 Check your email for an invite from Anthropic (subject: "You've been invited to Claude"). Click the link to create your account.
  • 2 Once logged in, go to console.anthropic.com and sign in.
  • 3 In the left sidebar, click API Keys.
  • 4 Click + Create Key, give it a name (e.g. radiantplay-claude-code), and click Create.
  • 5 Copy the key immediately — it is only shown once. Paste it into Notes or 1Password before closing the dialog.
⚠️ Your key starts with sk-ant-.... Keep it private — never share it or commit it to git.
3

Configure your API key and launch

One command sets the key, saves it permanently, configures Claude, and opens it

Replace your-api-key with the key you copied in Step 2, then run the whole thing:

Terminal — paste as one command
export ANTHROPIC_API_KEY="your-api-key" && \
echo "export ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY" >> ~/.zshrc && \
mkdir -p ~/.claude && \
echo '{"apiKeyHelper": "echo $ANTHROPIC_API_KEY"}' > ~/.claude/settings.json && \
claude
✅ After running this, Claude Code opens straight away — no separate launch step needed. Steps 3 and 4 are combined.
4

Verify Claude Code is working

Claude should already be open from Step 3 — just say hello to confirm

Claude Code opened at the end of Step 3. Type this in the session:

Claude Code session
> hello
# Claude responds — you're connected!
# Type /exit to close when done testing.
✅ If Claude responds, your key is working and you're all set. Jump to Step 5.

Troubleshooting

Symptom Fix
command not found: claude Shell hasn't picked up the new PATH yet. Run source ~/.zshrc or open a new terminal window and try again.
AuthenticationError / 401 API key is wrong or not exported. Verify with echo $ANTHROPIC_API_KEY.
npm install fails Check Node version — must be v18+. Run node --version.
rate_limit_error API usage limit reached — contact Mohammed to check quota.
Session opens, no response Check your internet connection. Claude Code requires outbound HTTPS.
5

Open RadiantPlay in Claude Code

Navigate to the project folder and start building
Option A — Terminal
Terminal
$ cd ~/code/radiantplay
$ claude
# Claude Code is now running inside the project
or
Option B — Finder right-click
  • 1 Open Finder and navigate to your radiantplay folder
  • 2 Right-click the folder → New Terminal at Folder
    If you don't see this, enable it in System Settings → Privacy & Security → Extensions → Finder Extensions
  • 3 In the terminal that opens, type claude and press Enter

Claude Code knows the repo structure, components, tokens, and all the rules. You work with it the same way you did with Cursor's chat panel — describe what you want to build and it will build it using Radiant components.

🎉 You're set up. Everything from here works the same as before. Use the slash commands below to start building.
Library updates

Keeping your library current

New components, tokens, and rules are pushed upstream regularly — here's how to stay in sync

The upstream RadiantPlay library is maintained centrally. Before starting new work — especially if your fork is from an older snapshot — run these steps to make sure you have the latest skills and components.

One-time setup

Get the latest skills

The .claude/commands/ folder holds all the skills Claude uses. If your fork is outdated, these may be missing or out of date. Run these four commands once to replace only the commands folder — your prototypes and all other files stay untouched.

Terminal — in your radiantplay folder
# 1. Add the upstream remote (skip if it already exists)
$ git remote add upstream https://galaxy.corp.thoughtspot.com/mohammed-faris/radiantplay.git

# 2. Fetch from upstream
$ git fetch upstream

# 3. Pull only the commands folder
$ git checkout upstream/main -- .claude/commands/

# 4. Restart Claude so it picks up the updated skills
$ claude
ℹ️ If you see error: remote upstream already exists on step 1, that's fine — skip it and continue from step 2.

Regular sync

Check what's changed in the library

Run a read-only diff of the upstream library — new components, token changes, rule updates. Nothing merges.

Claude Code session
/check-upstream
# Shows new components, token updates, and how many commits behind you are

Pull in the changes

Fetches, merges, auto-resolves the registry.ts conflict (your prototypes are preserved), verifies the build, and pushes to your fork. Your in-progress work is saved and restored automatically.

Claude Code session
/sync-upstream
# Saves WIP, merges, resolves conflicts, builds, pushes — all automatic
💡 When to sync: Run /check-upstream at the start of any new prototype sprint. If new components match what you're building, sync first — it saves you building something that already exists.
Slash commands

Skills & commands

Type these directly in the Claude Code terminal session
/new-prototype
Core
Start a new prototype
Asks discovery questions (goal, user, layout type), creates the branch, scaffolds files with the right layout template, generates a gallery thumbnail, and registers it automatically.
Layout-aware: Picks a dashboard, wizard, or single-page starter based on your answers. Creates styles.ts, mockData.ts, and components/ folder ready to go.
/radiant-check
New
Design system compliance check
Analyses your prototype across five dimensions: component coverage, custom components, token usage, layout patterns, and content guidelines. Shows a health score and easy wins.
Supports focus: Run /radiant-check content guidelines or /radiant-check visual styling to get a deep check on a specific area. Defaults to a full check.
/check-upstream
Sync
Preview library updates
Read-only check of what's new upstream — new components, token changes, rule updates, removed prototypes. Nothing is merged. Shows whether it's safe to sync or if there are conflicts to review.
When to use: At the start of a new prototype sprint, or whenever you want to see if new components are available that could help your work.
/sync-upstream
Sync
Pull in library changes
Fetches the latest components and tokens from the upstream library, auto-resolves the registry.ts conflict (your prototypes are preserved), verifies the build, and pushes to your fork.
Safe for WIP: Automatically saves and restores your in-progress work. No git knowledge needed — the whole process is handled for you.
/help
Core
Show all commands
Lists every available slash command with a short description. Good starting point if you're not sure what to run.
